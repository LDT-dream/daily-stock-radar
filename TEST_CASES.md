# daily-stock-radar 用例清单

每次修改后，逐项验证以下用例。

## 一、主页面（index.html）

### 1. 数据加载
- [ ] 页面打开后显示"数据更新: XXX"而不是"加载中..."
- [ ] 无 404 报错（data/latest.json 可访问）

### 2. 策略 Tab 切换
- [ ] 默认选中"全部"，显示所有策略的股票
- [ ] 点击"买贵的好公司"只显示 dongting_tech 策略的股票
- [ ] 点击"抄底便宜的公司"只显示 dongting_bottom 策略的股票
- [ ] Tab 上显示对应股票数量

### 3. 统计栏
- [ ] 显示股票总数、平均分、高/中/低分数量
- [ ] 切换 Tab 后统计数字跟随变化

### 4. 策略分组
- [ ] 每个策略有独立分组标题
- [ ] 分组标题显示股票数量、均分、高中低分分布
- [ ] 组内按分数降序排列

### 5. 股票卡片
- [ ] 显示股票名称、代码、分析时间（相对时间）
- [ ] 评分数字大且颜色正确（≥70绿、50-69黄、<50红）
- [ ] 操作建议标签颜色正确（买=绿、持=黄、卖=红）
- [ ] 如果与上次评分不同，显示 ▲/▼ 变化箭头
- [ ] 如果操作建议改变，显示 "观望 → 买入" 橙色标签

### 6. Sparkline 趋势线
- [ ] 有多次分析的股票卡片显示迷你趋势线
- [ ] 趋势线旁显示均值、最高、最低、分析次数
- [ ] 只有 1 次分析的股票不显示趋势线

### 7. 异常层高亮
- [ ] 与上次相比变化≥2分的维度显示为芯片标签
- [ ] 上升=默认色，下降=红色背景

### 8. 雷达图展开
- [ ] 点击"展开详情"显示雷达图和七层评分
- [ ] 雷达图 7 个维度标签正确
- [ ] 点击"收起详情"折叠回去

## 二、Modal 弹窗

### 9. 报告内容
- [ ] 点击卡片弹出 Modal，标题显示股票名+策略
- [ ] 报告是格式化的 Markdown（不是原始 JSON）
- [ ] 包含：综合评分、核心结论、分析摘要、七层评分表格
- [ ] ESC 或点击背景关闭 Modal

### 10. 评分趋势图
- [ ] 有多次分析的股票显示"评分趋势分析"
- [ ] 默认显示"综合评分"折线图
- [ ] 点击"七层评分"切换为多线图
- [ ] 图表可 hover 查看详细数据

### 11. 汇总统计
- [ ] 显示平均分、最高分、最低分、波动幅度、近期趋势
- [ ] 数值正确

### 12. 双次对比
- [ ] 历史记录每行有"对比"按钮
- [ ] 点击两次"对比"按钮后显示对比面板
- [ ] 对比面板显示两次评分、各维度差异条形图
- [ ] 再次点击取消选中

### 13. 历史时间线
- [ ] 每条记录显示日期、时间、评分、建议、七层色块
- [ ] 最新一条高亮显示
- [ ] 与上一次的分数变化显示正确

## 三、数据导出（export_radar.py）

### 14. 导出内容
- [ ] 每只股票有 raw_response（Markdown 格式，不是 JSON）
- [ ] 每只股票有 layer_scores（7层）
- [ ] 每只股票有 history 数组（多次分析记录）
- [ ] history 中每条有 score、advice、created_at、layer_scores

## 四、定时任务

### 15. 自动执行
- [ ] 定时任务到达时间后自动触发
- [ ] 分析完成后自动导出数据
- [ ] 导出后自动推送到 GitHub
- [ ] GitHub Pages 页面更新
